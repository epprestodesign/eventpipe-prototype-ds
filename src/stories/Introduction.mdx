# Eventpipe Design System

A **themed [Quasar](https://quasar.dev) component library** for **EventPipe** —
the platform teams use to manage **event hotel room blocks** — documented and
QA'd in Storybook.

EventPipe is a **B2B admin product**: event organizers, hotels, and companies
coordinate room blocks, pickup, reporting, and communications. This library is the
component system behind those screens — a Polaris-style set of function-grouped
components, the design tokens they carry, the app scaffolding they sit in, and
full page compositions built from them.

> **Building with an AI agent?** Read **Getting Started → Architecture &
> Conventions** first — it's the machine-oriented map (task → location,
> conventions, and the shared components to reuse instead of rebuild).

## Brand at a glance

- **Primary — Azure `#2561FA`** for actions, links, and selected states
  (`--ds-color-background-brand-bold`, `--ds-color-text-brand`).
- **Neutrals — Graphite** ramps (`--ds-palette-graphite-*`) for text, borders, and surfaces.
- **App chrome — EventPipe navy `#01113E`** for the sidebar / app bar; **accent green `#15AEB3`** for the logo and highlights.
- **Type — Product Sans** (system-font fallback stack).
- **Shape — uniform 4px radius**; **40px standard control height** for buttons and single-line fields.

## Components (grouped by function)

Every component carries the tokens above. Each story has a **Docs** tab with usage,
props, and guidance.

- **Actions** — triggers & overlays: Button, Button Group, Link, Dropdown, Menu,
  Popover, **Modal** (`DsModal`), Dialog, Backdrop, **Side Panel** (`DsSidePanel`).
- **Navigation** — moving through the app: Breadcrumbs, Tabs, Pagination, Stepper,
  **Search** (`DsSearch`).
- **Forms** — the **label-above** field system; every input composes into
  **Form Field** (`DsField`): Input (and Number / Payment / Date / File / Tags /
  Group), Pin Input, Text Field, Text Area, URL Field, Search, Autocomplete,
  Select, Multi-Select, Time Picker, Date Picker, Checkbox, Checkbox Tree, Radio
  Group, Switch, Choice Chips, Slider, Range, Transfer List, File Upload, Rating.
- **Feedback & Status** — communicating state: Badge, Badge Group, **Status**
  (`DsStatus`), Alert, Banner, Progress, Skeleton, Loading Indicator / States,
  Snackbar, **Toast**, Empty States.
- **Layout & Structure** — surfaces & structure: Box, Container, Grid, Stack,
  Divider, **Table**, **Card**, Card Headers, **Accordion**, Section Header,
  **List** / **List Item**, **Info Grid**, **Stat**.
- **Media & Visuals** — Avatar, Icon, Thumbnail, Image List, Carousel.
- **Typography & Content** — Text, Heading, Paragraph, Chip, Tooltip.

## Where to customize

| What you want to change | File |
| --- | --- |
| Brand colors, base font, global radius | `src/css/quasar.variables.scss` |
| Design tokens (palette, color roles, spacing, radius, type) | `src/css/ds-palette.scss`, `src/css/ds-color-tokens.scss`, `src/css/tokens.scss` |
| Per-component overrides & global CSS (incl. the 40px control height) | `src/css/app.scss` |
| Sidebar order / category IA | `.storybook/preview.js` (`storySort`) |
| Components | `src/components/` (`Ds*` primitives + `AppBar` / `AppShell`) |
| Stories | `src/stories/<category>/*.stories.js` |

## Workflow

1. `pnpm storybook` → open <a href="http://localhost:6006" target="_blank">localhost:6006</a>.
2. Edit any file above — the canvas hot-reloads instantly.
3. Use the **Controls** / **Docs** tabs on each story to QA props and variants.
4. When happy, `pnpm build-storybook` produces a static site in `storybook-static/`.

## Deploy

The build output is a plain static site. Pushing to `main` deploys to GitHub Pages
via `.github/workflows/deploy.yml` →
**https://epprestodesign.github.io/eventpipe-prototype-ds/**. See `README.md` for details.
