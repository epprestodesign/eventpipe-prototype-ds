# Architecture & Conventions

The machine-oriented map of the **Eventpipe Design System** — the component library
behind EventPipe's event hotel room-block admin product. If you're an **AI agent**
extending or editing the system, read this first: it tells you **where things live,
what to reuse, and the conventions to match** so new work is indistinguishable from
existing work.

---

## Reading the system efficiently

1. **Find the surface** in the sidebar — a token (`Foundations`), a reusable
   component (`Components / <function>`), a scaffold (`App Chrome`), or a full
   screen (`Pages`). See **Introduction** for the section map and **Components →
   Overview** for the function grouping.
2. **Locate the source** — every story maps to a component or page (tables below).
3. **Reuse before rebuilding** — most new screens compose existing `Ds*` primitives.
4. **Match the conventions** — tokens, naming, and story shape below.

## Design tokens & theming

- **Always use DS tokens**, never hardcoded hex. Tokens are CSS custom properties
  named `--ds-*`:
  - Palette ramps: `--ds-palette-<hue>-<step>` (e.g. `--ds-palette-azure-600`, `--ds-palette-graphite-200`).
  - Semantic roles: `--ds-color-text`, `--ds-color-text-subtle`, `--ds-color-surface`, `--ds-color-border`, `--ds-color-border-bold`, `--ds-color-background-brand-bold`, `--ds-color-text-danger`, `--ds-color-text-success`.
  - Scales: `--ds-radius-*`, `--ds-space-*`, `--ds-duration-fast`, `--ds-ease-standard`, `--ds-shadow-*`.
- **Primary brand = Azure `#2561FA`** — `--ds-color-background-brand-bold` for
  actions/selected, `--ds-color-text-brand` for brand text/links.
- **Neutrals = Graphite** (`--ds-palette-graphite-*`) for text, borders, surfaces.
- **App chrome = EventPipe navy `#01113E`** (sidebar / app bar); **accent green
  `#15AEB3`** for the logo and highlights.
- **Type = Product Sans** (`--ds-font-family`); **radius = uniform 4px**;
  **control height = 40px** for buttons and single-line fields.

## Component conventions

- **Vue 3 SFC, `<script setup>`.** Props via `defineProps`, events via
  `defineEmits`, imperative API via `defineExpose`.
- **Quasar is globally registered** — use `<q-*>` tags directly (no per-file
  import). Plugins available: `Notify`, `Dialog`, `Loading` (see `preview.js`).
- **Scoped styles**, BEM-ish naming: one **prefix per component**, then
  `prefix__element` / `prefix--modifier`. Teleported content (menus, notifies)
  needs an **unscoped** `<style>` block.
- **State sync** patterns in use: `v-model:<name>` (e.g. `v-model:open`),
  parent-owned reactive lists passed as props + `remove`/`update` events, and
  `defineExpose({ … })` for imperative APIs like `clear()`.
- **Overlays are primitives.** Centered dialogs use **`DsModal`** (backdrop, ESC,
  click-out, scroll-lock, mobile-fullscreen, `#header`/`#footer` slots, teleported
  to `<body>`); slide-overs use **`DsSidePanel`** (`side="right" / "left" /
  "center"`). Don't hand-roll overlay chrome.
- **Forms are label-above.** Every input composes into **`DsField`** (`DsInput`,
  `DsSelect`, etc.). The 40px control-height standard is applied globally in
  `app.scss`.
- Match the **comment density and idiom** of the file you're editing.

## Story conventions

- One `*.stories.js` per component. **`title` decides sidebar placement** — e.g.
  `title: 'Components/Forms/Input'`, `title: 'Pages/02 Events'`. The source folder
  is organizational only.
- Add `tags: ['autodocs']` for a generated Docs tab.
- **Variants are named exports** (`export const Default = …`). Full-page/scaffold
  components set `parameters: { layout: 'fullscreen' }`.
- Live/interactive demos use a `render()` with a `setup()` returning reactive state.
- Sidebar order is **explicit** in `preview.js` `storySort` (`Getting Started` →
  `Foundations` → `Components` → `App Chrome` → `Account` → `Pages` → `Design
  Requests` → `*`). Add new categories/pages there.

## Adding a component (checklist)

1. Create the SFC in `src/components/` (`Ds*` name for a primitive).
2. Use DS tokens + the prefix naming convention; reuse a shared primitive if one fits.
3. Add `src/stories/<category>/Name.stories.js` with a `Components/<function>/Name`
   `title`, `tags: ['autodocs']`, and variants.
4. If it's a new group/page, add it to `storySort` in `.storybook/preview.js`.
5. Verify with `pnpm build-storybook` (the deploy gate).

## Gotchas

- **Notify/menus are teleported** — style them from an **unscoped** `<style>` or
  global `app.scss` (e.g. `.ds-toast`).
- **Don't set `position` on a notification** (`.ds-toast`) — Quasar's leave
  transition sets it to `position:absolute`; an override (same specificity, loaded
  later) breaks the exit animation. Inset its progress bar to `0`.
- **40px control height vs floating labels** — the hard 40px applies to the DS
  label-above pattern. Fields that animate a label into the control (Quasar's
  floating label) keep their natural height; don't force 40px on those.
- **`references/` is local-only** (gitignored) — design screenshots live on disk, not in the repo.
- **Run from a git branch**, not `main`; deploy happens automatically on merge to `main`.
