# Bulk Reservation Edit — V1

Purpose-built **bulk management of (placeholder) staff-room reservations** for Team
Travel Source / Varsity. Resurrects the "Move" bulk-edit concept: select many
reservations, change their fields in bulk, confirm, and review the results — a task
the end-user portal can't do (no sort/filter/search, one reservation at a time).

Editing the **Hotel** field effectively *moves* a reservation, so a bulk hotel change
reads as *"moved to &lt;hotel&gt;."* All data in these screens is fake.

## ▶ Pilotable prototype (start to finish)

A standalone, clickable prototype is deployed alongside Storybook on GitHub Pages:

- **Live:** [https://epprestodesign.github.io/eventpipe-prototype-ds/prototype/](https://epprestodesign.github.io/eventpipe-prototype-ds/prototype/) → **Bulk Reservation Edit**
- **In Storybook:** the [Prototype — Full Flow](?path=/story/eventpipe-labs-bulk-reservation-edit--prototype) story runs the same self-contained component with no page reloads.
- **Local dev:** `pnpm dev:prototype` (serves the prototype app at `http://localhost:6007`).

The prototype and the story both mount `src/components/BulkReservationFlow.vue` — the
single design-system element that owns the whole flow.

## Phased rollout (from the brief)

1. **Phase 1 (V1):** edit **name** and **email** on placeholder reservations.
2. **Phase 2:** add check-in / check-out date changes.
3. **Phase 3:** add room changes.

The prototype exposes the full field set from the Figma so the whole flow is
pilotable; the phased scope governs what actually ships first.
